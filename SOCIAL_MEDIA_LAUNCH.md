## LinkedIn Post

**Title: Introducing IntentGuard: The AI Tool That Keeps Your Git Commits Honest**

Ever committed code with a message that didn't quite match what you actually did? We all have.

I'm excited to announce IntentGuard v0.2.0 - an open-source tool that uses AI to detect "semantic drift" between your commit messages and actual code changes.

**The Problem:**
- You write: "Add logging to user module"
- You actually: Modified database schemas, added API endpoints, and oh yeah, some logging

This drift accumulates into what we call "semantic debt" - a hidden cost that makes codebases harder to understand over time.

**The Solution:**
IntentGuard analyzes every commit using AI (supports OpenAI, Claude, and local models) and provides instant feedback:
- High alignment: ✅ Proceed
- Drift detected: ⚠️ Choose to abort, acknowledge debt, or proceed

**Key Innovation:**
The `.intentdebt.md` file tracks acknowledged drift, creating accountability and awareness. It's like a credit report for your commit hygiene.

**Get Started:**
```
npm install -g intentguard
```

Check it out: github.com/wiber/IntentGuard

## HackerNews Post

**Title:** IntentGuard - AI-powered semantic drift detection for git commits

**Text:**
Hi HN! I built IntentGuard to solve a problem we all face but rarely talk about: commit messages that don't match their code changes.

IntentGuard uses LLMs to analyze the semantic alignment between what you say you're doing and what you actually did. When it detects drift, you can choose to fix it or acknowledge the "semantic debt" for later.

What makes this different from a linter:
- It understands intent, not just syntax
- Tracks accumulated drift over time
- Never blocks commits (respects developer autonomy)
- Works with any LLM (OpenAI, Claude, local)

The killer feature is the `.intentdebt.md` file - a ledger of acknowledged misalignments that reminds you on future commits. It's fascinating to see how much drift accumulates in real projects.

Install: `npm install -g intentguard`
